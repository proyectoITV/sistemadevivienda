# SISTEMA DE CASCADA DIRECCIÓN → DEPARTAMENTO
## Implementación Completada

---

## 🎯 Objetivo Alcanzado

Implementar un sistema de filtrado en cascada donde:
1. Usuario selecciona una **DIRECCIÓN**
2. Sistema carga automáticamente los **DEPARTAMENTOS** de esa dirección
3. Usuario selecciona el departamento deseado

**Resultado:** Menor cantidad de errores, mejor UX, validación en cliente

## 📊 Flujo de Datos

### Crear Empleado (Nuevo)
```
1. Pagina carga
   ├─ Dirección: dropdown vacío con todas las direcciones
   └─ Departamento: disabled

2. Usuario selecciona dirección
   ├─ onChange → JavaScript dispara AJAX
   └─ GET /api/departamentos-por-direccion/?iddireccion=1

3. Server responde con JSON
   ├─ Lista de departamentos de esa dirección
   └─ JavaScript actualiza el select

4. Usuario selecciona departamento
   └─ Formulario listo para enviar
```

### Editar Empleado (Existente)
```
1. Página carga
   ├─ Dirección: pre-cargada con la dirección del empleado
   ├─ Departamento: pre-cargado con el departamento del empleado
   └─ __init__ detecta iddepartamento y extrae la dirección

2. Al inicializar el select de dirección
   ├─ JavaScript dispara onChange automáticamente
   └─ Carga los departamentos de esa dirección

3. Se preserva el departamento seleccionado
   └─ Usuario puede modificar o dejar igual
```
